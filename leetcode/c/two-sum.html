Two Sum - C Solution

LeetCode Problem #1

1. Two Sum

Easy

Problem Description

Given an array of integers nums and an integer target, return indices of the two numbers such that they add up to target.

You may assume that each input would have exactly one solution, and you may not use the same element twice.

You can return the answer in any order.

Examples

Example 1:

Input: nums = [2,7,11,15], target = 9

Output: [0,1]

Explanation: Because nums[0] + nums[1] == 9, we return [0, 1].

Example 2:

Input: nums = [3,2,4], target = 6

Output: [1,2]

Example 3:

Input: nums = [3,3], target = 6

Output: [0,1]

Constraints

  • 2 ≤ nums.length ≤ 104
  • -109 ≤ nums[i] ≤ 109
  • -109 ≤ target ≤ 109
  • Only one valid answer exists.

Solution

Approach 1: Brute Force

The simplest approach is to check every possible pair of numbers in the array.

/**
 * Note: The returned array must be malloced, assume caller calls free().
 */
int* twoSum(int* nums, int numsSize, int target, int* returnSize) {
    *returnSize = 2;
    int* result = (int*)malloc(2 * sizeof(int));
    
    for (int i = 0; i < numsSize; i++) {
        for (int j = i + 1; j < numsSize; j++) {
            if (nums[i] + nums[j] == target) {
                result[0] = i;
                result[1] = j;
                return result;
            }
        }
    }
    
    // No solution found (should not reach here based on problem constraints)
    return result;
}
Time Complexity

O(n²) - We have two nested loops, each going through the array.

Space Complexity

O(1) - We only use a constant amount of extra space.

Approach 2: Hash Table

We can use a hash table to improve the time complexity. For each element, we check if the complement (target - nums[i]) exists in the hash table. If it does, we've found our solution. If not, we add the current element to the hash table.

#include <stdlib.h>
#include <string.h>

// Simple hash table implementation
typedef struct {
    int key;   // The number
    int value; // Its index in the array
    int used;  // Whether this slot is used
} HashEntry;

typedef struct {
    HashEntry* entries;
    int size;
} HashMap;

// Initialize hash map
HashMap* createHashMap(int size) {
    HashMap* map = (HashMap*)malloc(sizeof(HashMap));
    map->size = size;
    map->entries = (HashEntry*)malloc(sizeof(HashEntry) * size);
    memset(map->entries, 0, sizeof(HashEntry) * size);
    return map;
}

// Hash function
int hash(int key, int size) {
    // Simple hash function for demonstration
    return abs(key) % size;
}

// Put key-value pair in hash map
void put(HashMap* map, int key, int value) {
    int index = hash(key, map->size);
    
    // Linear probing to handle collisions
    while (map->entries[index].used && map->entries[index].key != key) {
        index = (index + 1) % map->size;
    }
    
    map->entries[index].key = key;
    map->entries[index].value = value;
    map->entries[index].used = 1;
}

// Get value by key, returns -1 if not found
int get(HashMap* map, int key) {
    int index = hash(key, map->size);
    
    // Linear probing to find the key
    while (map->entries[index].used) {
        if (map->entries[index].key == key) {
            return map->entries[index].value;
        }
        index = (index + 1) % map->size;
        
        // If we've checked all entries, break
        if (index == hash(key, map->size)) {
            break;
        }
    }
    
    return -1; // Not found
}

// Free hash map
void freeHashMap(HashMap* map) {
    free(map->entries);
    free(map);
}

/**
 * Note: The returned array must be malloced, assume caller calls free().
 */
int* twoSum(int* nums, int numsSize, int target, int* returnSize) {
    *returnSize = 2;
    int* result = (int*)malloc(2 * sizeof(int));
    
    // Create a hash map with size twice the array size to reduce collisions
    HashMap* map = createHashMap(numsSize * 2);
    
    for (int i = 0; i < numsSize; i++) {
        int complement = target - nums[i];
        
        // Check if complement exists in hash map
        int complementIndex = get(map, complement);
        if (complementIndex != -1) {
            result[0] = complementIndex;
            result[1] = i;
            
            // Free the hash map before returning
            freeHashMap(map);
            return result;
        }
        
        // Add current number to hash map
        put(map, nums[i], i);
    }
    
    // Free the hash map
    freeHashMap(map);
    
    // No solution found (should not reach here based on problem constraints)
    return result;
}
Time Complexity

O(n) - We only need to traverse the array once.

Space Complexity

O(n) - In the worst case, we might need to store all elements in the hash table.

Notes

  • The hash table approach is much more efficient for large arrays.
  • In C, implementing a hash table requires more code compared to languages with built-in hash table data structures like Python or Java.
  • The hash table implementation above is simplified and not optimized for all cases.
  • For very small arrays (n < 10), the brute force approach might be faster due to lower constant factors.
