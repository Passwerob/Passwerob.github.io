Problem 1464: Maximum Product of Two Elements in an Array

1464. Maximum Product of Two Elements in an Array

Easy

Given the array of integers nums, you will choose two different indices i and j of that array. Return the maximum value of (nums[i]-1)*(nums[j]-1).

Examples:

Example 1:
Input: nums = [3,4,5,2] Output: 12 Explanation: If you choose the indices i=1 and j=2 (indexed from 0), you will get the maximum value, that is, (nums[1]-1)*(nums[2]-1) = (4-1)*(5-1) = 3*4 = 12.
Example 2:
Input: nums = [1,5,4,5] Output: 16 Explanation: Choosing the indices i=1 and j=3 (indexed from 0), you will get the maximum value of (5-1)*(5-1) = 16.
Example 3:
Input: nums = [3,7] Output: 12

Constraints:

  • 2 <= nums.length <= 500
  • 1 <= nums[i] <= 10^3

Solution

Approach 1: Sorting

A straightforward approach is to sort the array and then compute the product of the two largest elements minus 1.

Python 
class Solution:
    def maxProduct(self, nums: List[int]) -> int:
        nums.sort()
        return (nums[-1] - 1) * (nums[-2] - 1)

Complexity Analysis:

Time Complexity: O(n log n) due to sorting.
Space Complexity: O(1) or O(n) depending on the sorting implementation.

Approach 2: Find Two Largest Elements

We can also find the two largest elements in a single pass through the array.

Python 
class Solution:
    def maxProduct(self, nums: List[int]) -> int:
        first_max = second_max = 0
        
        for num in nums:
            if num > first_max:
                second_max = first_max
                first_max = num
            elif num > second_max:
                second_max = num
                
        return (first_max - 1) * (second_max - 1)

Complexity Analysis:

Time Complexity: O(n) where n is the length of the array.
Space Complexity: O(1) as we only use two variables.

Time Complexity: O(n) where n is the length of the array.

Notes:

For this problem, Approach 2 is the most efficient as it has O(n) time complexity and O(1) space complexity. My solution works but is more complex and uses more memory than necessary.

Since the problem states that all elements are positive (1 <= nums[i] <= 10^3), we don't need to handle negative numbers separately.
